The code above demonstrate a couple properties of the Clone Of expression:

+ The expression result is the same as the input object. This can be seen by the use of an [anchored type](https://github.com/dlebansais/Easly-Language/blob/master/Doc/Nodes/Type/AnchoredType.md).
+ The expression result is a new object. The [New expression](https://github.com/dlebansais/Easly-Language/blob/master/Doc/Nodes/Expression/NewExpression.md) is therefore a valid assertion (although in this example, the value of `Clone` is lost upon return).

If the source object is of a type that allows instances to be compared (as most do), objects `P` and `Clone` in the example above would be equal.

Only instances of a class can be cloned. You can't clone an agent or a tuple for example. But you can clone an instance of a class with [value semantic](https://github.com/dlebansais/Easly-Language/blob/master/Doc/Nodes/Class.md#reference-type).

Note that creating deep clones is a recursive operation: the whole tree of objects is cloned. This accounts for cyles in references.

## Cloning multiple objects

If the source expression has more than one result, and as long as each of them is an instance of a class, the Clone Of expression also has multiple results, each of them the clone of their corresponding instance.

Since expression results can have names (for example `Result`), the Clone Of expression preserves these names. 

# Translation to C&#35;

In C#, cloning is a per-class concept, and obtained by implementing (and referencing) the `ICloneable` interface. Only classes for which at least one instance was cloned need to implement the interface.

```csharp
public object Clone()
{
    using (MemoryStream ms = new MemoryStream())
    {
        ISerializer s = new Serializer();
        s.Serialize(ms, this);
        ms.Seek(0, SeekOrigin.Begin);
        return s.Deserialize(ms);
    }
}
```
  
This implementation performs a deep clone. A shallow clone can be obtained directly from the .NET method `MemberwiseClone`. Note that an intermediate method is necessary since `MemberwiseClone` is a protected method of `Object`. 

```csharp
public object CloneShallow()
{
    return MemberwiseClone();
}
```

Either of these features can then be called where the clone of expression is used:

```csharp
object Clone = Foo.Clone();
```

```csharp
object Clone = Foo.CloneShallow();
```
